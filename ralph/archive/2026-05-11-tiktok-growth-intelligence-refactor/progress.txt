  - Validation: `npm run build`, `npm run test:unit`.
  - Learning: a score layered on top of raw deltas gives better “what to repeat next” behavior than sorting by a single metric.

  - Extended the dashboard payload with lifetime aggregates, period boundaries, profile period metrics, daily series, and richer post analytics.

  - Validation: `npm run build`, `npm run test:unit`, `npm run test:e2e`.
  - Learning: keeping fallback states explicit avoided misleading period numbers while preserving a usable dashboard for new profiles.

  - Refactored the dashboard UI to distinguish current base metrics from growth-in-window metrics.

  - Expanded unit and Playwright fixtures to exercise the new payload shape end to end.
  - Validation: `npm run build`, `npm run test:unit`, `npm run test:e2e`.
  - Learning: once the payload semantics were correct, the UI became much simpler to reason about and the copy no longer had to compensate for backend ambiguity.
